주차는 로봇을 움직이기 전, 로봇의 몸(하드웨어)을 이해하고 로봇이 활동할 **공간(컴퓨터 환경)**을 만드는 준비 단계입니다. 2주차에서 겪으셨던 "파일 형식"이나 "환경 설정" 문제들이 여기서 완벽히 정리되어야 합니다.

비프로그래머를 위한 1주차 스텝-바이-스텝 가이드입니다.

1단계: 로봇의 구조 이해하기 (하드웨어)


코딩하기 전에 내가 조종할 로봇이 어떻게 생겼는지 알아야 합니다.
머리(Head): 카메라가 들어있어 앞을 볼 수 있고, 목 관절을 통해 상하좌우로 움직입니다.
안테나(Antennas): 머리 위에 두 개가 있으며, 감정을 표현하는 데 핵심적인 역할을 합니다.
바디(Body): 로봇의 몸체로, 전체적인 중심을 잡고 회전(body_yaw)할 수 있습니다.
좌표계: 로봇은 $x$(앞뒤), $y$(좌우), $z$(높이) 좌표를 사용합니다.
예: "왼쪽을 봐"라고 명령하는 대신 "$y$축으로 -10만큼 움직여"라고 명령하게 됩니다.

2단계: 로봇의 공부방 만들기 (환경 설정)
컴퓨터에 Reachy Mini 전용 작업실을 차리는 과정입니다. 
이 과정은 **CMD(명령 프롬프트)**에서 진행합니다.
1. 파이썬 설치: 컴퓨터가 파이썬 언어를 알아듣게 합니다. (설치 시 'Add Python to PATH' 꼭 체크!)
2. 가상환경 생성: 로봇 조종 도구들이 다른 프로그램과 섞이지 않게 전용 방을 만듭니다.

Bash
python -m venv reachy_env

방 입장(활성화):
Bash
reachy_env\Scripts\activate
입력 후 줄 맨 앞에 (reachy_env)가 생겼는지 확인하세요.

조종 도구(SDK) 설치:
Bash
pip install reachy-mini
pip install reachy-mini[mujoco]
mujoco는 화면에 가상 로봇(시뮬레이션)을 띄우기 위한 도구입니다.

3단계: 로봇 시동 걸기 (Daemon 실행)로봇 프로그래밍에서 가장 중요한 개념인 **데몬(Daemon)**을 실행합니다. 데몬은 코드가 보낸 신호를 로봇 모터의 움직임으로 바꿔주는 '번역기'입니다.CMD 창에서 입력:
Bash
reachy-mini-daemon --sim
성공 결과: MuJoCo라는 이름의 창이 뜨면서 귀여운 Reachy Mini 로봇이 화면에 나타납니다.주의: 이 창(CMD)은 로봇이 켜져 있는 상태이므로, 공부가 끝날 때까지 절대로 끄지 마세요.

4단계: 첫 번째 인사 코드 (VS Code)이제 VS Code를 열고 로봇과 연결이 잘 되었는지 확인하는 코드를 작성합니다.VS Code에서 week1_test.py 파일을 만듭니다. (끝에 .py 꼭 확인!)아래 코드를 복사해서 붙여넣습니다.Pythonfrom reachy_mini import ReachyMini # 조종 도구 가져오기

# 로봇과 대화를 시도합니다.
with ReachyMini() as mini:
    print("Reachy Mini 연결 성공!")
    
    # 현재 로봇의 각 관절 상태가 어떤지 물어봅니다.
    state = mini.get_state()
    print(f"로봇의 현재 상태: {state}")

실행: VS Code 상단의 실행 버튼을 누릅니다.확인: 하단 터미널 창에 "연결 성공!"과 함께 복잡한 숫자들(로봇 상태)이 출력되면 1주차 목표 달성입니다!
